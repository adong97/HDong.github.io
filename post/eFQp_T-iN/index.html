一、单机 Redis 业务容器化

在k8s环境中运⾏redis服务

1.1 准备基础镜像

~#pwd
/opt/k8s-data/dockerfile/web/magedu/redis

~# tree .
.
├── Dockerfile
├── build-command.sh
├── redis-4.0.14.tar.gz
├── redis.conf
└── run_redis.sh

0 directories, 5 files

编辑Dockerfile

vim Dockerfile

#Redis Image
FROM harbor.weixd.com/baseimage/magedu-centos-base:7.9.2009

MAINTAINER weixd "weixdself@163.com"

ADD redis-4.0.14.tar.gz /usr/local/src
RUN ln -sv /usr/local/src/redis-4.0.14 /usr/local/redis && cd /usr/local/redis && make && cp src/redis-cli /usr/sbin/ && cp src/redis-server  /usr/sbin/ && mkdir -pv /data/redis-data
ADD redis.conf /usr/local/redis/redis.conf
ADD run_redis.sh /usr/local/redis/run_redis.sh

EXPOSE 6379

CMD ["/usr/local/redis/run_redis.sh"]

在构建镜像得时候，会将redis.conf配置文件添加到镜像当中。因此，需要提前做好配置优化，包括快照触发参数等

vim redis.conf 

save 900 1
save 5 1
save 300 10
save 60 10000

requirepass 123456  #配置redis密码

在Dockerfile中添加run_redis.sh文件，主要是为了将redis跑起来

vim run_redis.sh

#!/bin/bash
/usr/sbin/redis-server /usr/local/redis/redis.conf  #启动redis服务
tail -f  /etc/hosts  # 作为守护进程

执行构建脚本

vim build-command.sh

#!/bin/bash
TAG=$1
docker build -t harbor.weixd.com/magedu/redis:${TAG} .
sleep 3
docker push  harbor.weixd.com/magedu/redis:${TAG}

##开始构建镜像
bash build-command.sh 4.0.14

测试redis镜像是否可用

~# docker run -it -p 6379:6379 harbor.weixd.com/magedu/redis:4.0.14

2.2 K8s应用 redis 服务

基于PV/PVC保存数据，实现k8s中运⾏Redis服务

创建PV

vim redis-persistentvolume.yaml

---
apiVersion: v1
kind: PersistentVolume
metadata:
  name: redis-datadir-pv-1
  namespace: magedu
spec:
  capacity:
    storage: 10Gi
  accessModes:
    - ReadWriteOnce
  nfs:
    path: /data/k8s/magedu/redis-datadir-1
    server: 10.1.13.186

~# kubectl apply -f redis-persistentvolume.yaml

创建PVC

vim redis-persistentvolumeclaim.yaml

---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: redis-datadir-pvc-1
  namespace: magedu
spec:
  volumeName: redis-datadir-pv-1
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi

~# kubectl apply -f redis-persistentvolumeclaim.yaml

创建redis服务

vim redis.yaml

kind: Deployment
#apiVersion: extensions/v1beta1
apiVersion: apps/v1
metadata:
  labels:
    app: devops-redis
  name: deploy-devops-redis
  namespace: magedu
spec:
  replicas: 1
  selector:
    matchLabels:
      app: devops-redis
  template:
    metadata:
      labels:
        app: devops-redis
    spec:
      containers:
        - name: redis-container
          image: harbor.weixd.com/magedu/redis:4.0.14
          imagePullPolicy: Always
          volumeMounts:
          - mountPath: "/data/redis-data/"
            name: redis-datadir
      volumes:
        - name: redis-datadir
          persistentVolumeClaim:
            claimName: redis-datadir-pvc-1

---
kind: Service
apiVersion: v1
metadata:
  labels:
    app: devops-redis
  name: srv-devops-redis
  namespace: magedu
spec:
  type: NodePort
  ports:
  - name: http
    port: 6379
    targetPort: 6379
    nodePort: 30379
  selector:
    app: devops-redis
  sessionAffinity: ClientIP
  sessionAffinityConfig:
    clientIP:
      timeoutSeconds: 10800


~# kubectl apply -f redis.yaml

2.2 创建redis.conf文件

redis cluster配置文件,以configmap得形式给集群提供。

~# cat redis.conf

appendonly yes   #打开aof文件
cluster-enabled yes #开启集群，必须打开
cluster-config-file /var/lib/redis/nodes.conf  #集群配置文件，自动生成
cluster-node-timeout 5000  #集群初始化超时时间
dir /var/lib/redis  #数据目录
port 6379  #端口

2.3 部署redis cluster

基于redis.conf创建cpnfigmap

~# kubectl -n  magedu create configmap redis-conf --from-file=redis.conf
configmap/redis-conf created

~# kubectl -n magedu describe configmaps redis-conf

部署redis集群

~# vim  redis.yaml

## 这个SVC用于K8S内部访问
apiVersion: v1
kind: Service
metadata:
  name: redis
  namespace: magedu
  labels:
    app: redis
spec:
  selector:
    app: redis
    appCluster: redis-cluster
  ports:
  - name: redis
    port: 6379
  clusterIP: None

---
apiVersion: v1
kind: Service
metadata:
  name: redis-access
  namespace: magedu
  labels:
    app: redis
spec:
  selector:
    app: redis
    appCluster: redis-cluster
  ports:
  - name: redis-access
    protocol: TCP
    port: 6379
    targetPort: 6379

---
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: redis
  namespace: magedu
spec:
  serviceName: redis
  replicas: 6
  selector:
    matchLabels:
      app: redis
      appCluster: redis-cluster
  template:
    metadata:
      labels:
        app: redis
        appCluster: redis-cluster
    spec:
      terminationGracePeriodSeconds: 20
      affinity:
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 100
            podAffinityTerm:
              labelSelector:
                matchExpressions:
                - key: app
                  operator: In
                  values:
                  - redis
              topologyKey: kubernetes.io/hostname
      containers:
      - name: redis
        image: harbor.weixd.com/magedu/redis:5.0.14
        command:
          - "redis-server"
        args:
          - "/etc/redis/redis.conf"  #配置文件
          - "--protected-mode"  # 不打开安全模式
          - "no"
        resources:
          requests:
            cpu: "500m"
            memory: "500Mi"
        ports:
        - containerPort: 6379
          name: redis
          protocol: TCP
        - containerPort: 16379
          name: cluster
          protocol: TCP
        volumeMounts:
        - name: conf
          mountPath: /etc/redis
        - name: data
          mountPath: /var/lib/redis
      volumes:
      - name: conf
        configMap:
          name: redis-conf
          items:
          - key: redis.conf
            path: redis.conf
  volumeClaimTemplates:  #通过PVC模板文件创建
  - metadata:
      name: data
      namespace: magedu
    spec:
      accessModes: [ "ReadWriteOnce" ]  #权限，每个PVC以读写得方式被单个挂载
      resources:
        requests:
          storage: 5Gi

创建redis pod和svc
~# kubectl apply -f redis.yaml
~# kubectl -n magedu get pod -o wide
