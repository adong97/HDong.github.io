2.2 高可用负载均衡

部署节点：
k8s-master01: 10.1.13.186
k8s-master02: 10.1.13.187

2.2.1 安装Haproxy

1. 安装haproxy
~# apt install haproxy -y

2. 修改配置文件
~# vim /etc/haproxy/haproxy.cfg
...  ##最后添加listen
listen k8s-api-6443
  bind 10.1.13.199:6443  #绑定VIP地址的6443
  mode tcp
  server master1 10.1.13.186 check inter 3s fall 3 rise 1
  server master2 10.1.13.187 check inter 3s fall 3 rise 1
...

3.重启服务并设置为开启自启
~# systemctl restart haproxy.service && systemctl enable haproxy.service

4.检测服务时候监听6443端口
~# ss -ntlp
State         Recv-Q         Send-Q                 Local Address:Port                 Peer Address:Port        Process                                           
LISTEN        0              489                      10.1.13.199:6443                      0.0.0.0:*            users:(("haproxy",pid=15601,fd=7)) 

5.拷贝文件到另一台服务器，并重启服务
~# scp /etc/haproxy/haproxy.cfg 10.1.13.187:/etc/haproxy/
~# systemctl restart haproxy

重启发现haproxy服务启动失败，原因是：
haproxy监听VIP的6443端口，但是VIP现在飘在master01节点上，所以master02节点的haproxy起不来
可以通过`journalctl -u haproxy`查看详细日志。

因此需要我们优化系统内核参数 (两个负载均衡节点都做变更)
~# sysctl -a |grep local #过滤系统参数
net.ipv4.ip_nonlocal_bind = 0  表示不允许应用程序监听在本机没有的IP地址上，需要改为1.

~# vim /etc/sysctl.conf
net.ipv4.ip_nonlocal_bind = 1

~# sysctl -p ##生效配置文件
~# systemctl restart haproxy  ##重启haproxy服务,即可
~# systemctl enable haproxy

2.2.2 安装keepalived

1. 安装keepalived
~# apt install keepalived -y

2.拷贝模板文件
~# cp /usr/share/doc/keepalived/samples/keepalived.conf.vrrp /etc/keepalived/keepalived.conf

3. 修改配置文件
~# vim /etc/keepalived/keepalived.conf
vrrp_instance VI_1 {
    state MASTER
    interface ens160       ##更改网卡名称，与实际相符
    garp_master_delay 10
    smtp_alert
    virtual_router_id 51
    priority 100           ##优先级
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 1111
    }
    virtual_ipaddress {
        10.1.13.199 dev ens160 lable eth160:0   ##添加多个VIP，方便后面使用
        10.1.13.200 dev ens160 lable eth160:1
        10.1.13.201 dev ens160 lable eth160:2
    }
}

#重启服务
systemctl restart keepalived  

#测试VIP是否可用
ping 10.1.13.199

4. 拷贝keepalived.conf到另一台服务器，修改配置
~# scp /etc/keepalived/keepalived.conf 10.1.13.187:/etc/keepalived/keepalived.conf
~# vim /etc/keepalived/keepalived.conf
vrrp_instance VI_1 {
    state BACKUP   ##更改角色
    interface ens160
    garp_master_delay 10
    smtp_alert
    virtual_router_id 51
    priority 80   ##修改优先级
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass 1111
    }
    virtual_ipaddress {
        10.1.13.199 dev ens160 lable eth160:0
        10.1.13.200 dev ens160 lable eth160:1
        10.1.13.201 dev ens160 lable eth160:2
    }
}

5.启动服务，并设置为开机自启
~# systemctl start keepalived && systemctl enable keepalived

2.3.3 安装harbor

1. 解压harbor二进制文件到/opt目录
~# wget  https://github.com/goharbor/harbor/releases/download/v2.4.2/harbor-offline-installer-v2.4.2.tgz
~# tar -zxvf harbor-offline-installer-v2.4.2.tgz -C /opt

2.自签发https证书
~# mkdir certs
~# openssl genrsa -out /opt/harbor/certs/harbor-ca.key  ##生成私有key
~# openssl req -x509 -new -nodes -key /opt/harbor/certs/harbor-ca.key -subj "/CN=harbor.weixd.com" -days 7120 -out /opt/harbor/certs/harbor-ca.crt  ##签证

3.修改配置文件
~# cd /opt/harbor/
~# cp harbor.yml.tmpl harbor.yml
~# vim harbor.yml
hostname: harbor.weixd.com   ##修改域名或IP地址
https:
  # https port for harbor, default is 443
  port: 443
  # The path of cert and key files for nginx
  certificate: /opt/harbor/certs/harbor-ca.crt ##修改公钥存放路径
  private_key: /opt/harbor/certs/harbor-ca.key ##修改私钥存放路径
  
harbor_admin_password: Harbor12345  ##harbor的登录密码

data_volume: /data  ##数据存储目录

4.安装harbor
./install.sh --help ##查看安装参数

Please set --with-trivy if needs enable Trivy in Harbor  ##镜像安全扫描
Please set --with-chartmuseum if needs enable Chartmuseum in Harbor ##支持helm的使用

## 安装
./install.sh --with-trivy --with-chartmuseum

5.在电脑本地hosts配置文件中添加IP地址解析
10.1.13.187 harbor.weixd.com

6.页面访问测试
harbor.weixd.com

三、kubeasz安装k8s

github项目地址：https://github.com/easzlab/kubeasz

- 注意1：确保各节点时区设置一致、时间同步。 如果你的环境没有提供NTP 时间同步，推荐集成安装[chrony](https://github.com/easzlab/kubeasz/blob/master/docs/guide/chrony.md)
- 注意2：确保在干净的系统上开始安装，不要使用曾经装过kubeadm或其他k8s发行版的环境
- 注意3：建议操作系统升级到新的稳定内核，请结合阅读[内核升级文档](https://github.com/easzlab/kubeasz/blob/master/docs/guide/kernel_upgrade.md)
- 注意4：在公有云上创建多主集群，请结合阅读[在公有云上部署 kubeasz](https://github.com/easzlab/kubeasz/blob/master/docs/setup/kubeasz_on_public_cloud.md)

2. forward # 表示解析不了的域名，往哪个地址转发.默认转发地址:/etc/resolv.conf

3.3.1 Coredns配置详解

vim coredns.yaml
## Corefile:  为coredns的核心配置段

data:
  Corefile: |
    .:53 {
        errors   ## 错误信息标准输出
        health { ## 检测服务是否存活,在CoreDNS的http://localhost:8080/health端口提供CoreDNS服务的健康报告
            lameduck 5s
        }
        ready ## CoreDNS监听8181端口,当coreDNS的插件都已就绪时,访问该端口会返回200 OK
        kubernetes cluster.local in-addr.arpa ip6.arpa {  ## 
            pods insecure
            fallthrough in-addr.arpa ip6.arpa
            ttl 30
        }
        prometheus :9153
        forward . /etc/resolv.conf {
            max_concurrent 1000
        }
        cache 30
        loop
        reload
        loadbalance
    }
        mysever.online {
          forward . 
        } 
    
######### 参数解析：
errors  ## 错误信息标准输出

health  ## 检测服务是否存活,在CoreDNS的http://localhost:8080/health端口提供CoreDNS服务的健康报告

ready   ## CoreDNS监听8181端口,当coreDNS的插件都已就绪时,访问该端口会返回200 OK

kubernetes  ## CoreDNS将基于kubernetes service name 进行DNS查询并返回查询记录给客户端

prometheus ## CoreDNS的度量指标数据以Prometheus的key-value的格式在http://localhost:9153/metrics URL上提供

forward  ## 不是Kubernetes集群内的其他任何域名查询都将转发到预定义的目的server,如(/etc/resolv.conf或IP(如8.8.8.8))

cache ## 启用service解析缓存,单位为秒

loop ## 检测域名解析是否有死循环,如coredns转发给内网服务器,而内网DNS服务器又转发给coredns,如果发现转发是死循环,则强制中止CoreDNS进程(kubernetes会重建)

reload ## 检测corefile是否更改,在重新编辑configmap配置后,默认2分钟会优雅的自动加载

loadbalance ## 轮询DNS域名解析,如果一个域名存在多个记录则轮询解析

自定义添加域名解析示例：
  mysever.online {  ## 域名为mysever.online
          forward . 172.16.16.16:53 ##将域名转发到172.16.16.16:53解析
  } ## 写在Corefile段外
