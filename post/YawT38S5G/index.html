K8s业务迁移: 部署Jenkins

2022-06-19 / 4 min read

一、运行Java应用

基于java命令，运⾏java war包或jar包，本次以jenkins.war 包部署⽅式为例，且要求jenkins的数据保存⾄外部存储(NFS或者PVC)，其他java应⽤看实际需求是否需要将数据保存⾄外部存储。

1.1 构建基础镜像

镜像目录文件

 ~# pwd
/opt/k8s-data/dockerfile/web/magedu/jenkins

# tree .
.
├── Dockerfile
├── build-command.sh
├── jenkins-2.319.2.war
└── run_jenkins.sh

0 directories, 4 files

构建Jenkins镜像

# cat Dockerfile
#Jenkins Version 2.190.1
FROM harbor.weixd.com/pub-image/jdk-base:v8.212

MAINTAINER weixd wxdongself@163.com

ADD jenkins-2.319.2.war /apps/jenkins/jenkins.war
ADD run_jenkins.sh /usr/bin/


EXPOSE 8080

CMD ["/usr/bin/run_jenkins.sh"]

启动脚本
# cat run_jenkins.sh
#!/bin/bash
cd /apps/jenkins && java -server -Xms1024m -Xmx1024m -Xss512k -jar jenkins.war --webroot=/apps/jenkins/jenkins-data --httpPort=8080


# cat build-command.sh
#!/bin/bash
docker build -t  harbor.weixd.com/magedu/jenkins:v2.319.2 .
echo "镜像制作完成，即将上传至Harbor服务器"
sleep 1
docker push harbor.weixd.com/magedu/jenkins:v2.319.2
echo "镜像上传完成"

执行构建Jenkins
# bash build-command.sh

验证Jenkins镜像

# docker run -it --rm -p 8088:8080 harbor.weixd.com/magedu/jenkins:v2.319.2

1.2 运行Jenkins应用

  1. 创建PV/PVC
    需要两个PVC，⼀个保存jenkins的数据，⼀个保存.jenkins的数据。
创建数据目录
# mkdir -p /data/k8s/magedu/{jenkins-data,jenkins-root-data}


vim jenkins-persistentvolume.yaml
---
apiVersion: v1
kind: PersistentVolume
metadata:
  name: jenkins-datadir-pv
  namespace: magedu
spec:
  capacity:
    storage: 100Gi
  accessModes:
    - ReadWriteOnce
  nfs:
    server: 10.1.13.186
    path: /data/k8s/magedu/jenkins-data

---
apiVersion: v1
kind: PersistentVolume
metadata:
  name: jenkins-root-datadir-pv
  namespace: magedu
spec:
  capacity:
    storage: 100Gi
  accessModes:
    - ReadWriteOnce
  nfs:
    server: 10.1.13.186
    path: /data/k8s/magedu/jenkins-root-data


vim jenkins-persistentvolumeclaim.yaml
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: jenkins-datadir-pvc
  namespace: magedu
spec:
  volumeName: jenkins-datadir-pv
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 80Gi

---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: jenkins-root-data-pvc
  namespace: magedu
spec:
  volumeName: jenkins-root-datadir-pv
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 80Gi

创建PV/PVC

# kubectl apply -f jenkins-persistentvolume.yaml

# kubectl apply -f jenkins-persistentvolumeclaim.yaml

# kubectl -n magedu get pv,pvc
  1. 部署Jenkins服务
vim  jenkins.yaml

kind: Deployment
#apiVersion: extensions/v1beta1
apiVersion: apps/v1
metadata:
  labels:
    app: magedu-jenkins
  name: magedu-jenkins-deployment
  namespace: magedu
spec:
  replicas: 1
  selector:
    matchLabels:
      app: magedu-jenkins
  template:
    metadata:
      labels:
        app: magedu-jenkins
    spec:
      containers:
      - name: magedu-jenkins-container
        image: harbor.weixd.com/magedu/jenkins:v2.319.2
        #imagePullPolicy: IfNotPresent
        imagePullPolicy: Always
        ports:
        - containerPort: 8080
          protocol: TCP
          name: http
        volumeMounts:
        - mountPath: "/apps/jenkins/jenkins-data/"
          name: jenkins-datadir-magedu
        - mountPath: "/root/.jenkins"
          name: jenkins-root-datadir
      volumes:
        - name: jenkins-datadir-magedu
          persistentVolumeClaim:
            claimName: jenkins-datadir-pvc
        - name: jenkins-root-datadir
          persistentVolumeClaim:
            claimName: jenkins-root-data-pvc

---
kind: Service
apiVersion: v1
metadata:
  labels:
    app: magedu-jenkins
  name: magedu-jenkins-service
  namespace: magedu
spec:
  type: NodePort
  ports:
  - name: http
    port: 80
    protocol: TCP
    targetPort: 8080
    nodePort: 30880
  selector:
    app: magedu-jenkins

部署Jenkins

# kubectl apply -f jenkins.yaml
