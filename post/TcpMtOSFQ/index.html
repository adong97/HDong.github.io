ETCD常用操作

拷贝etcdctl命令行工具：

#将Etcd容器中的etcdctl命令拷贝到本机/usr/bin/etcdctl
$ docker cp etcd_container:/usr/local/bin/etcdctl /usr/bin/etcdctl

#如果etcd没有使用Docker容器，则将Etcd安装目录下的etcdctl拷贝到/usr/bin/etcdctl目录下
$ cp /opt/etcd/bin/etcdctl /usr/bin/etcdctl

查看etcd集群的成员节点：

#声明通过ETCDCTL_API=3这个版本拿取数据
$ export ETCDCTL_API=3

$ etcdctl --endpoints=https://[127.0.0.1]:2379 \
              --cacert=/etc/kubernetes/pki/etcd/ca.crt \
              --cert=/etc/kubernetes/pki/etcd/healthcheck-client.crt \
              --key=/etc/kubernetes/pki/etcd/healthcheck-client.key member list -w table

## --endpoints指定要访问哪个ETCD的server
## 2379为客户端连接端口，2380为service端通信端口
## --cacert指定根证书
## --cert根据根证书签发的证书	
## --key签发证书的私钥

#设置别名，方便操作
$ alias etcdctl='etcdctl --endpoints=https://127.0.0.1:2379 --cacert=/etc/kubernetes/pki/etcd/ca.crt --cert=/etc/kubernetes/pki/etcd/healthcheck-client.crt --key=/etc/kubernetes/pki/etcd/healthcheck-client.key'

#查看ETCD集群成员节点
$ etcdctl member list -w table

查看etcd集群节点状态：

$ etcdctl endpoint status -w table

$ etcdctl endpoint health -w table

设置key值:

#设置值
$ etcdctl put luffy 1

#查询值
$ etcdctl get luffy

#我们可以按照目录结构的方式存储数据，这样的好处就是，组件存储数据可以按照目录区分，不会重复
$ etcdctl put /luffy/key1 val1

查看所有key值：

$  etcdctl get / --prefix --keys-only
## --prefix 前缀，把所有使用/前缀的key读出来
## --keys-only 只取key值

###取值含义
示例：
/registry/services/specs/kube-system/kubelet
> /registry 第一层，固定的
> /services 第二层，资源类型

查看具体的key对应的数据：

$ etcdctl get /registry/pods/jenkins/sonar-postgres-7fc5d748b6-gtmsb

list-watch:

$ etcdctl watch /luffy/ --prefix
$ etcdctl put /luffy/key1 val1

添加定时任务做数据快照（重要！）

$ etcdctl snapshot save `hostname`-etcd_`date +%Y%m%d%H%M`.db

恢复快照：

  1. 停止etcd和apiserver

  2. 移走当前数据目录

    $ mv /var/lib/etcd/ /tmp

  3. 恢复快照

    $ etcdctl snapshot restore `hostname`-etcd_`date +%Y%m%d%H%M`.db --data-dir=/var/lib/etcd/

  4. 集群恢复

    https://github.com/etcd-io/etcd/blob/master/Documentation/op-guide/recovery.md

  • namespace删除问题

    很多情况下，会出现namespace删除卡住的问题，此时可以通过操作etcd来删除数据：

    #查询数据
$ etcdctl get /registry/namespace --prefix	--keys-only

#删除数据，确保namespace下没有pod等数据
$ etcdctl del /registry/namespace/luffy
